## 1. 구현 내용

### 1.1 Fragment 유사도 검색 에이전트 (`src/agent.py`)
- LangGraph StateGraph 기반 4단계 워크플로우
  - `parse_input`: 사용자 입력 SMILES 파싱 및 유효성 검증
  - `load_data`: QM8 데이터셋 로드 (21,787 molecules)
  - `search_similar`: 선택된 방법(들)로 fragment 유사도 검색 수행
  - `format_output`: 결과 포맷팅 (단일 방법 / 비교 모드 자동 전환)
- CLI 지원: `--methods`, `--top-k`, `--no-substruct-filter` 옵션
- 에러 핸들링 (유효하지 않은 SMILES 처리)

### 1.2 유사도 검색 엔진 (`src/similarity.py`) — 6가지 방법 지원

| 방법 | 설명 | 원리 |
|------|------|------|
| `morgan` | Morgan (ECFP4) FP + Tanimoto | 각 원자 주변 반경 2의 환경을 해시하여 2048bit 벡터 생성. 원자 환경 패턴을 비교 |
| `maccs` | MACCS Keys (166bit) + Tanimoto | 사전 정의된 166개 구조 키(작용기, 고리 등)의 존재 여부를 비트로 표현 |
| `rdkit` | RDKit FP + Tanimoto | 분자 그래프 위의 경로(path)를 열거하여 2048bit 벡터 생성 |
| `atompair` | AtomPair FP + Tanimoto | 모든 원자 쌍의 (원자 종류, 거리) 조합을 인코딩 |
| `torsion` | Topological Torsion FP + Tanimoto | 연속 4개 원자 경로(torsion angle 위상)를 인코딩 |
| `mcs` | Maximum Common Substructure | 두 분자의 최대 공통 부분구조 원자수 / 큰 쪽 원자수 |

- 검색 전략: **Substructure 필터링** (fragment 포함 분자만 추출) → **유사도 계산 및 정렬**
- 유사도 점수: 0.0(완전 다름) ~ 1.0(동일)

### 1.3 방법론 비교 문서 (`docs/similarity_search_methods.md`)
- 유사도 검색 방식 비교: Fingerprint, Substructure, GNN, Graph RAG, MCS
- Graph RAG 장/단점 상세 분석
- 재료 분야 최근 기술 동향 (Foundation Models, LLM+Chemistry, Inverse Design, Vector DB)
- 향후 확장 전략 (CSV → PostgreSQL → Vector DB/Graph RAG)

## 4. 테스트 결과

### 4.1 테스트 설명

**테스트 목적**: 동일한 fragment SMILES를 입력했을 때, 6가지 유사도 방법이 각각 어떤 분자를 높은 순위로 반환하는지, 유사도 점수 분포는 어떻게 다른지, 소요시간은 얼마인지를 비교한다.

**테스트 방법**:
1. QM8 데이터셋(21,787개 분자)에서 fragment를 부분구조로 포함하는 분자를 먼저 필터링 (Substructure Match)
2. 필터링된 후보 분자들에 대해 6가지 방법으로 각각 유사도를 계산
3. 유사도 높은 순으로 정렬하여 상위 3개(Top-3) 결과를 비교

**결과 테이블 읽는 법**:
- **행(row)**: 각 유사도 방법 (morgan, maccs, rdkit, atompair, torsion, mcs)
- **Top-N 유사도**: 해당 방법으로 계산한 유사도 순위에서 N번째 분자의 유사도 점수
  - 1.0 = fragment와 완전 동일한 구조, 0.0 = 전혀 유사하지 않음
- **소요시간**: 후보 분자 전체에 대해 유사도를 계산하고 정렬하는 데 걸린 시간 (초)

---

### 4.2 벤젠 fragment (`c1ccccc1`) — 방향족 고리

**테스트 의도**: 벤젠 고리(6각 방향족 고리)를 fragment로 입력하여, 벤젠을 포함하는 분자 중 구조적으로 가장 유사한 것을 찾는다.

- 후보 분자 수: **42개** (21,787개 중 벤젠 고리를 포함하는 분자)

#### 방법별 Top-3 상세 결과

**Morgan (ECFP4)**:

| 순위 | SMILES | 유사도 | 분자 설명 |
|------|--------|--------|----------|
| 1 | `[H]c1c([H])c([H])c([H])c([H])c1[H]` | 1.0000 | 벤젠 그 자체 |
| 2 | `[H]c1c([H])c([H])c(C([H])([H])[H])c([H])c1[H]` | 0.2727 | 톨루엔 (메틸벤젠) |
| 3 | `[H]c1c([H])c([H])c(N([H])[H])c([H])c1[H]` | 0.2727 | 아닐린 (아미노벤젠) |

**MACCS Keys**:

| 순위 | SMILES | 유사도 | 분자 설명 |
|------|--------|--------|----------|
| 1 | `[H]c1c([H])c([H])c([H])c([H])c1[H]` | 1.0000 | 벤젠 그 자체 |
| 2 | `[H]c1c([H])c([H])c(C([H])([H])[H])c([H])c1[H]` | 0.7500 | 톨루엔 |
| 3 | `[H]C#Cc1c([H])c([H])c([H])c([H])c1[H]` | 0.7500 | 페닐아세틸렌 |

**MCS (최대 공통 부분구조)**:

| 순위 | SMILES | 유사도 | 분자 설명 |
|------|--------|--------|----------|
| 1 | `[H]c1c([H])c([H])c([H])c([H])c1[H]` | 1.0000 | 벤젠 그 자체 |
| 2 | `[H]c1c([H])c([H])c(C([H])([H])[H])c([H])c1[H]` | 0.8571 | 톨루엔 |
| 3 | `[H]c1c([H])c([H])c(N([H])[H])c([H])c1[H]` | 0.8571 | 아닐린 |

#### 방법 간 비교 요약

| 방법 | Top-1 유사도 | Top-2 유사도 | Top-3 유사도 | 소요시간(초) |
|------|-------------|-------------|-------------|-------------|
| morgan | 1.0000 | 0.2727 | 0.2727 | 0.0003 |
| maccs | 1.0000 | 0.7500 | 0.7500 | 0.0001 |
| rdkit | 1.0000 | 0.3243 | 0.3158 | 0.0002 |
| atompair | 1.0000 | 0.4444 | 0.4211 | 0.0001 |
| torsion | 1.0000 | 0.2222 | 0.2222 | 0.0001 |
| mcs | 1.0000 | 0.8571 | 0.8571 | 0.0043 |

**분석**:
- Top-1은 모든 방법에서 벤젠 자체 (유사도 1.0)
- **MCS**가 Top-2 이하에서 가장 높은 유사도 점수(0.857)를 부여 → 공통 부분구조 비율을 직접 측정하므로, 벤젠 고리를 공유하는 톨루엔/아닐린이 높게 평가됨
- **MACCS Keys**도 높은 점수(0.75) → 166개 구조 키 중 벤젠 관련 키가 상당수 겹침
- **Morgan**은 낮은 점수(0.27) → ECFP는 치환기까지 포함한 원자 환경을 세밀하게 구분하므로, 치환기가 달라지면 점수가 크게 떨어짐
- 속도는 모든 fingerprint 방법이 0.0001~0.0003초로 매우 빠르고, MCS만 0.004초로 상대적으로 느림

### 4.3 카르보닐 fragment (`C=O`) — 작용기

**테스트 의도**: 카르보닐기(C=O)는 알데히드, 케톤, 카르복시산, 아미드 등 다양한 작용기의 핵심 구조이다. 이 작은 fragment로 검색 시 각 방법이 어떻게 반응하는지 비교한다.

- 후보 분자 수: **7,544개** (전체의 약 35%가 C=O를 포함)

#### 방법 간 비교 요약

| 방법 | Top-1 유사도 | Top-2 유사도 | Top-3 유사도 | 소요시간(초) |
|------|-------------|-------------|-------------|-------------|
| morgan | 1.0000 | 0.1429 | 0.1333 | 0.0245 |
| maccs | 1.0000 | 0.5000 | 0.5000 | 0.0251 |
| rdkit | 1.0000 | 0.3333 | 0.3333 | 0.0256 |
| atompair | 1.0000 | 0.0667 | 0.0667 | 0.0236 |
| torsion | **0.0000** | 0.0000 | 0.0000 | 0.0235 |
| mcs | 1.0000 | 0.6667 | 0.6667 | 0.0067 |

**분석**:
- **Torsion이 전부 0.0** → Topological Torsion은 연속 4개 원자 경로를 인코딩하는데, C=O는 원자 2개뿐이라 torsion 패턴 자체가 생성되지 않음. **작은 fragment에는 부적합한 방법**
- **MCS**가 가장 직관적 → 포름알데히드(H₂C=O)는 원자 2개, 아세트알데히드(CH₃CHO)는 원자 3개이므로 MCS 유사도 = 2/3 = 0.6667
- **MACCS**도 비교적 높은 점수(0.50) → 카르보닐 관련 구조 키가 공유됨
- 후보가 7,544개로 많아 소요시간이 벤젠(42개) 대비 ~100배 증가했으나, 여전히 0.03초 미만

### 4.4 질소 fragment (`N`) — 최소 단위 원자

**테스트 의도**: 단일 질소 원자(N)를 fragment로 입력하여 가장 작은 단위의 검색 동작을 확인한다.

- 후보 분자 수: **12,675개** (전체의 약 58%가 질소를 포함)

#### MCS 단일 방법 결과

| 순위 | SMILES | 유사도 | 분자 설명 |
|------|--------|--------|----------|
| 1 | `[H]N([H])[H]` | 1.0000 | 암모니아 (NH₃) |
| 2 | `[H]C#N` | 0.5000 | 시안화수소 (HCN) |
| 3 | `[H]C([H])([H])C#N` | 0.3333 | 아세토니트릴 (CH₃CN) |

**분석**:
- MCS 유사도가 직관적으로 해석됨: NH₃는 질소 1개/전체 1개 = 1.0, HCN은 질소 1개/전체 2개 = 0.5
- 단일 원자 fragment이므로 fingerprint 기반 방법들은 매우 낮은 유사도를 반환 (morgan Top-2: 0.0625)
- 후보가 12,675개로 가장 많지만 검색 성능에 문제 없음

### 4.7 방법별 특성 종합

아래 표는 3가지 fragment 테스트를 통해 관찰된 각 방법의 실제 동작 특성이다.

| 방법 | 점수 분포 | 작은 fragment 대응 | 속도 | 적합한 사용 사례 |
|------|----------|-------------------|------|-----------------|
| **morgan** | 점수 차이가 큼 (Top-1과 Top-2 간 갭이 큼) | 사용 가능 | 매우 빠름 | 치환기까지 엄밀히 구분하고 싶을 때 |
| **maccs** | 점수가 비교적 높게 분포 | 사용 가능 | 매우 빠름 | 작용기 수준의 유사도를 빠르게 평가할 때 |
| **rdkit** | morgan과 maccs의 중간 | 사용 가능 | 매우 빠름 | 범용적 유사도 평가 |
| **atompair** | 점수 차이가 큼 | 사용 가능 | 매우 빠름 | 원자 쌍 거리 관점의 유사도가 필요할 때 |
| **torsion** | 큰 분자에서만 의미 있음 | **원자 4개 미만 fragment에서 0.0** | 매우 빠름 | 분자 골격 형태 비교 (큰 fragment 전용) |
| **mcs** | 가장 직관적 (공통 구조 비율) | 우수 | 상대적으로 느림 | 화학적으로 해석 가능한 유사도가 필요할 때 |
